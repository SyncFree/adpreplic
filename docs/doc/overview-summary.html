The system is composed of the Strategy Layer, the Replication Layer and Database (DB). The Strategy Layer is responsible to identify what Data Centre (DC) to replicate the data and for how long, i.e. when to remove a replica and from which DC. The Replication Layer is responsible to the eventual consistency between the different replicas in each DC.

Startegy Layer

It is responsable to state when data is replicated, for how long and where. Some strategies are:
  • No Replication where the data is only kept in one DC, the one it was created on.
  • Full Replication where a copy of the data exists in all DCs. DC what implement this type of strategy are Antidote and Riak Enterprise.
  • Adaptive Replication where the data is replicated in certain DCs depending of their pattern of acces and modification. An exmaple implementation of thi strategy is provided here.

Replication Layer

It is responsible to propagate the changes to all the other DCs with replicas.

Database (DB)

In principal the idea is that the DB can be any of the existing DB, e.g. Riak, Oracle...
